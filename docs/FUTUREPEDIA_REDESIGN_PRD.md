# Agentic AI Tools 重构 PRD

## 1. 项目背景

当前站点更像内容落地页集合，首页与工具目录主要承担 SEO 承接功能，缺少目录型产品最关键的能力：

- 用户无法高效发现和筛选工具
- 没有收藏、投稿、订阅等持续留存机制
- 页面结构以静态文案为主，缺少“可更新的数据层”
- 所谓“工具目录”仍是硬编码内容，编辑效率低

对标 Futurepedia 这类成熟产品，重构后的站点应从“AI 工具文章站”升级为“AI 工具发现平台”，保留 SEO 文章资产，但把首页、目录页和投稿漏斗改成以目录体验为核心。

## 2. 产品目标

### 2.1 业务目标

- 提升工具发现效率，增加目录页停留时长与点击率
- 构建可持续更新的数据层，降低新增工具的发布成本
- 增加用户留存入口：收藏、邮件订阅、工具投稿
- 为后续商业化预留位：赞助位、精选推荐、分类赞助、线索收集

### 2.2 用户目标

- 快速找到适合自己角色与场景的 AI 工具
- 通过筛选、排序与编辑推荐降低选择成本
- 保存待评估工具，形成个人 shortlist
- 提交新工具或申请收录，参与平台生态

## 3. 核心用户

### 3.1 Builder / Developer

- 关注代码、Agent、自动化、API 生态
- 希望按类别、价格、能力标签快速筛选

### 3.2 Marketer / Operator

- 关注内容、视频、增长、自动化、知识管理工具
- 更看重上手速度、价格区间、典型用例

### 3.3 Creator / Solo Founder

- 关注内容生产、品牌、视频、语音、设计、效率工具
- 需要编辑推荐和场景化集合，而不是纯参数列表

## 4. 问题定义

当前版本存在以下关键问题：

1. 首页信息架构错误
   首页仍以长文内容逻辑组织，不是目录产品入口。

2. 目录不可维护
   工具列表硬编码在 HTML 中，无法通过后端数据动态更新。

3. 缺少用户动作闭环
   没有收藏、投稿、订阅等中间层行为，只有“访问文章”。

4. 品牌感与产品感不足
   视觉方向偏旧式 SEO 内容站，不像可信的产品型目录平台。

## 5. 产品原则

- 目录优先：先帮助用户发现，再引导用户阅读深度内容
- 编辑策展优先于堆量：突出精选、趋势、角色场景
- 数据可维护：前台列表必须由后端数据驱动
- SEO 与产品兼容：首页/目录页服务人，详情页继续服务搜索
- 留存动作前置：收藏、订阅、投稿必须进入一层信息架构

## 6. 版本范围

## 6.1 V1 范围

- 新首页
- 新工具目录页
- 新工具投稿页
- Express API
- JSON 数据层
- 收藏能力
- 邮件订阅能力
- 投稿入库能力
- 分类统计与动态筛选

## 6.2 V1 不做

- 登录/注册
- 真正的支付系统
- CMS 后台
- 评论系统
- 向量搜索/推荐算法
- 自动抓取外部工具库

## 7. 信息架构

### 7.1 首页

- Hero：搜索、CTA、价值主张
- Trending now：高热度精选工具
- Category grid：按分类发现
- Editorial picks：按角色场景策展
- Saved shortlist：用户收藏入口
- Newsletter：周报订阅
- Submit CTA：投稿与商务入口

### 7.2 工具目录页

- 搜索框
- 类别筛选
- 定价筛选
- 已验证筛选
- 排序
- 动态结果数
- 工具卡片
- 快速保存
- 快速访问官网
- 快速进入站内评测

### 7.3 投稿页

- 收录价值说明
- 审核流程
- 投稿表单
- 提交成功反馈

## 8. 前端功能需求

### 8.1 首页功能

- 支持全站级搜索跳转到目录页
- 展示工具总数、分类数、已验证工具数
- 支持从首页直接收藏精选工具
- 展示用户已收藏工具数量
- Newsletter 表单支持直接提交到后端

### 8.2 目录页功能

- 输入关键词后实时筛选
- 按类别筛选
- 按价格模型筛选
- 按“已验证”筛选
- 排序支持：
  - Featured first
  - Trending
  - Top rated
  - Recently reviewed
  - Alphabetical
- 筛选状态可映射到 URL，支持分享与回访
- 卡片支持收藏/取消收藏
- 展示工具标签、价格、评分、编辑点评
- 支持工具详情快速查看区
- 支持空状态与重置筛选

### 8.3 投稿页功能

- 校验工具名、URL、分类、联系人邮箱
- 提交后返回成功提示
- 表单数据写入后端状态文件

## 9. 后端功能需求

### 9.1 API 能力

- `GET /api/health`
- `GET /api/home`
- `GET /api/categories`
- `GET /api/tools`
- `GET /api/tools/:slug`
- `GET /api/favorites`
- `POST /api/favorites/toggle`
- `POST /api/newsletter`
- `POST /api/submissions`

### 9.2 数据模型

工具数据字段：

- slug
- name
- category
- categoryLabel
- subcategory
- description
- editorialNote
- pricingModel
- pricingLabel
- rating
- trendingScore
- verified
- featured
- sponsored
- tags
- useCases
- websiteUrl
- reviewUrl
- updatedAt

状态数据字段：

- favoritesByClientId
- newsletterLeads
- submissions

### 9.3 持久化策略

- 工具目录使用 `data/tools.json`
- 用户轻量状态使用 `data/app-state.json`
- 收藏通过 `clientId` 维持
- 适用于单机部署与 MVP 验证

## 10. 关键用户流程

### 10.1 发现流程

首页搜索 -> 跳转目录页 -> 筛选结果 -> 收藏/访问官网/阅读评测

### 10.2 留存流程

浏览精选工具 -> 收藏若干 -> 订阅周报 -> 后续回访继续比较

### 10.3 供给流程

工具方进入投稿页 -> 填写信息 -> 写入 submissions -> 后续人工审核

## 11. 指标

### 11.1 北极星指标

- 每周收藏工具数

### 11.2 支撑指标

- 首页到目录页点击率
- 目录页工具卡点击率
- 收藏转化率
- Newsletter 提交率
- 投稿提交数
- 已验证工具占比

## 12. 验收标准

1. 首页、目录页、投稿页全部由新信息架构驱动
2. 目录列表不再硬编码在 HTML 中
3. 本地运行时可通过 API 返回真实数据
4. 收藏、订阅、投稿能写入状态层
5. UI 与旧版长文式首页有明显产品形态差异

## 13. 后续迭代方向

- 登录后跨设备同步收藏
- 后台审核与工具管理面板
- 分类页与专题页自动生成
- 更细粒度的角色/场景推荐
- 商务合作与赞助位管理
